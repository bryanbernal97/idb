About

Twitch.tv is a website where streamers live stream themselves providing content centered around their lives and video games. There are many facets of this experience that we define as models: Streamer, Game, Community, Team. A streamer plays different games, takes part in different established communities centered around these games, and often join various esports teams which their streams then represent. Included with all of them are many other media and information which enhances each of those models. Twitch contains all of this information but there isn’t a good place that brings all of this information together in an easy to navigate environment. We decided to make this information more readily available by linking these data points together on a website.  We are accomplishing this by fleshing out different user use cases, scraping available RESTful API for Twitch and other supplementary data, defining important models which will connect to each other, and exploring new tools to make this process easier and more intuitive. We have named our website StreamGlean.me (glean - extract (information) from various sources) to represent the process of gathering stream information.

Tools

Apiary

We used Apiary to document the main API we used for this first part of the project.

Trello

We used Trello to do issue tracking as a team.

PlanITpoker

We used PlanITpoker to come up with user stories and estimate track their estimated and actual completion times.

Namecheap

We used Namecheap to register our custom domain.

AWS

We used AWS to host our website. We used AWS sub-product Elastic Beanstalk to connect our website development with git and to deploy versions of our website. We used AWS sub-product EC2 for different instnaces to allow for a dev and prod environment, and we used AWS sub-product S3 to store our uploaded website content.
